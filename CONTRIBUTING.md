# Contributing

Thank you for your interest in contributing!

## Development Setup

1. Clone the repository
2. Read `CLAUDE.md` for project conventions and architecture
3. Read `forge.toml` for build/test commands

## Development Workflow

- **main is sacred.** All changes go through PRs. No direct pushes.
- Branch naming: `feat/`, `fix/`, `chore/`, `docs/` prefixes
- Conventional commits: `type: description` format
- PRs require CI passing

## Pull Request Process

1. Create a branch from `main`
2. Make your changes
3. Run the test suite
4. Open a PR with a clear description
5. Address review feedback

## Autonomous Development

This project uses an autonomous development pipeline. Issues labeled `backlog/ready`
are automatically picked up and implemented by AI agents. See `docs/internal/autodev-pipeline.md`
for details.

## Code of Conduct

Be respectful. Be constructive. Be kind.
